# ==================== Format ====================
# type: description
# ================================================

# ==================== Types ====================
# feat: 新機能
# update: 機能改善
# fix: バグ修正
# style: コードの意味に影響しない変更（空白、書式、セミコロンの欠落など）
# refactor: バグの修正でも機能の追加でもないコード変更
# docs: ドキュメントのみの変更
# perf: パフォーマンスを向上
# test: 不足しているテストの追加や既存のテストの修正
# chore: ciやbuildや補助ツールなどのsrcやテストファイルを変更しないその他の変更
# remove: ファイル/コード削除
# revert: 以前のコミットを取り消す
# bump: バージョンアップ
# ================================================
